# Test Plan — Chubaryan University Schedule Planner

## 1. Цели тестирования
Цель тестирования — убедиться, что приложение:
- корректно генерирует расписание с соблюдением всех жёстких constraints;
- корректно обрабатывает пользовательские данные (группы, преподаватели, аудитории);
- стабильно работает в интерфейсе JavaFX;
- обеспечивает отсутствие конфликтов (аудиторий, преподавателей, групп);
- позволяет вручную корректировать расписание без ошибок;
- корректно экспортирует данные в PDF, Excel и HTML.

## 2. Объекты тестирования
- ScheduleGeneratorService
- ConstraintService
- ScheduleEditorController (UI)
- ImportExportService
- Генетический алгоритм, Tabu Search
- Модуль визуализации расписания

## 3. Виды тестирования
- **Функциональное**
- **Пользовательское (UX-тестирование)**
- **Регрессионное** (локальное)
- **Smoke-тестирование**

## 4. Окружение
- ОС: Windows 11 / macOS Ventura
- JDK 17
- JavaFX 17
- Maven 3.9.x
- PostgreSQL 14 (тестовая база)
- Тестовые CSV/Excel-файлы для импорта

## 5. Инструменты
- JUnit 5
- TestFX (UI-тесты)
- Excel/Google Sheets для тест-кейсов
- Google Forms для сбора отзывов
- https://forms.gle/dZVvUoGtRXrMqGyB8

## 6. Критерии входа
- Проект успешно собирается (`mvn clean compile`)
- Все зависимости установлены
- GUI запускается

## 7. Критерии выхода
- 95% функциональных тестов пройдены
- Все критические ошибки исправлены
- Сформирован test_report и issues.md
- Опрос пользователей завершён

## 8. Риски
- Неполные тестовые данные
- Ошибки алгоритмов оптимизации
- Нагрузочные тесты невозможны в рамках задания
